## Main Update: Modpacks

You can now publish **modpacks** on Modifold — collections of mods bundled into a single archive (RAR / ZIP). You can upload them, and other users will be able to download and install them.

What already works:

- A dedicated modpacks section next to mods
- A **Modpack** project type in the project creation modal
- Separate tags for modpacks
- ... otherwise, the same features as a **Mod** project type

What doesn’t work:

- We don’t know yet, but we’ll be grateful if you report any bugs and improvements you find for this new project type.

Please note: **the maximum size of a single version file is 100 MB**. We’ll be raising this limit in upcoming updates.

## What’s New in the UI

Highlights:

- A new tag selector in project settings: a list of icon buttons without a dropdown popover
- Some improvements on the search page: a **Modpacks** tab and the correct indicator
- Updated background for the mods and modpacks sections
- Redesigned notifications page
- Logged-out users can now change the site language and theme
- Fixes and improvements to the mobile layout

## Other Improvements

In parallel, we shipped a lot of small but important updates:

- Updated the settings pages (project and profile) and their sidebars
- Added an extra account protection method: 2FA authorization
- Improved the project comments page
- Adjusted the profile verification request page
- Organizations can now add links to their social networks
- Moved a person’s social links out of the profile and into a separate block on the page
- Removed some scripts that were adding extra page load time
- On the profile page, clicking a person’s follower or following count opens a modal with the users they follow / who follow them

## What’s Next

We’re continuing to develop modpacks and improve platform stability. We welcome your criticism and feedback — thanks to you, we keep striving to be better.

Thank you for building Modifold with us.
